# 메디스파크 랜딩페이지 Apps Script 명세서

## 1. 목적

- 랜딩페이지에서 수집한 상담 데이터를 Google Apps Script Web App으로 전송한다.
- 전송된 데이터는 Google Sheets에 자동 저장한다.
- 내부 운영용으로 `개발자정보: 유정희`를 함께 기록한다.

## 2. 연동 구조

1. 사용자가 랜딩페이지에서 `상담하기` 클릭
2. 이름, 연락처 입력 후 폼 제출
3. 프런트엔드가 Apps Script Web App URL로 `POST` 요청 전송
4. Apps Script가 데이터 유효성 검사
5. Google Sheets 마지막 행에 데이터 추가
6. 성공 JSON 응답 반환
7. 프런트엔드가 완료 메시지 표시

## 3. 전송 방식

- Method: `POST`
- Content-Type: `application/json`
- Endpoint 예시:
  - `https://script.google.com/macros/s/YOUR_DEPLOYMENT_ID/exec`

## 4. 필수 전송 데이터

- `name`: 고객명
- `phone`: 연락처
- `developer`: 고정값 `유정희`

## 5. 선택 전송 데이터

- `interestType`: 관심 타입
- `inquiry`: 문의 내용
- `visitDateTime`: 방문 희망 일시
- `privacyConsent`: 개인정보 동의 여부
- `pageName`: 유입 페이지명
- `projectName`: 프로젝트명
- `status`: 기본값 `신규접수`

## 6. 요청 JSON 예시

```json
{
  "name": "홍길동",
  "phone": "01012345678",
  "interestType": "84A",
  "inquiry": "분양가 문의",
  "visitDateTime": "2026-06-28 14:00",
  "privacyConsent": true,
  "pageName": "메디스파크 랜딩페이지",
  "projectName": "브레인시티 메디스파크 로제비앙 모아엘가",
  "developer": "유정희",
  "status": "신규접수"
}
```

## 7. 응답 형식

### 성공

```json
{
  "ok": true,
  "message": "접수가 완료되었습니다. 담당자가 확인 후 순차적으로 연락드립니다."
}
```

### 실패

```json
{
  "ok": false,
  "message": "전송 중 문제가 발생했습니다. 다시 시도해주세요."
}
```

## 8. 검증 규칙

- `name`은 공백 제거 후 1자 이상
- `phone`은 숫자만 남긴 뒤 10~11자리
- `developer`는 비어 있으면 안 됨
- `privacyConsent`는 `true`여야 저장 가능하도록 운영 가능

## 9. 시트 저장 컬럼 순서

1. 등록일시
2. 이름
3. 연락처
4. 관심타입
5. 문의내용
6. 방문희망일시
7. 개인정보동의여부
8. 유입페이지명
9. 프로젝트명
10. 개발자정보
11. 상담상태
12. utm_source
13. utm_medium
14. utm_campaign
15. utm_content
16. utm_term
17. referrer
18. landing_path
19. user_agent

## 10. Apps Script 설정값

- Spreadsheet ID: 운영 시트 ID 입력
- Sheet Name: 예시 `leads`
- Timezone: `Asia/Seoul`

## 11. 배포 방식

1. Google Sheets 생성
2. `확장 프로그램 > Apps Script` 열기
3. 제공된 `Code.gs` 붙여넣기
4. `SPREADSHEET_ID`, `SHEET_NAME` 수정
5. `배포 > 새 배포 > 웹 앱`
6. 실행 사용자: 본인
7. 접근 권한: 링크가 있는 모든 사용자
8. 배포 URL을 HTML의 `SCRIPT_URL`에 입력

## 12. 운영 주의사항

- 테스트 중복 입력이 많이 쌓일 수 있으니 시트에 상태 컬럼을 둔다.
- 광고 매체별 분리가 필요하면 추후 `campaign`, `source`, `medium` 컬럼을 추가한다.
- 개인정보 수집 문구와 보관 기간은 실제 운영 정책에 맞춰 최종 검수한다.
- 허니팟 필드, 최소 체류시간, 동일 번호 중복 접수 제한을 기본 스팸 방어로 사용한다.
