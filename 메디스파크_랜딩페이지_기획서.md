# 브레인시티 메디스파크 분양 홍보 랜딩페이지 기획서

## 1. 기획 목적

- 제공된 PDF와 이미지 자료를 바탕으로 `브레인시티 메디스파크 로제비앙 모아엘가`의 분양 홍보용 랜딩페이지 구조를 설계한다.
- 참조 사이트 [m-bee.kr/braincity](https://m-bee.kr/braincity)의 짧고 강한 전환 구조를 반영하되, 메디스파크의 입지와 미래가치가 더 선명하게 보이도록 한 단계 업그레이드한다.
- 일반적인 정보 나열형 분양 페이지가 아니라, 방문자가 이탈하지 않고 `관심 → 확신 → 문의`로 자연스럽게 이동하도록 터널 마케팅 방식으로 구성한다.
- 최종 목표는 `상담 신청`, `방문예약`, `관심고객 등록` 전환 확보이다.
- 대표 문의번호는 `010-6689-2348`로 통일하여 모든 CTA와 고정 상담 영역에 반복 노출한다.
- 랜딩페이지 개발 담당자는 `유정희`로 지정하며, 고객 DB 수집 시 개발자 정보도 함께 저장되도록 설계한다.

## 2-1. 참조 페이지 분석 요약

### 참조 페이지에서 가져올 요소

- 첫 화면에서 바로 `단지명`, `브랜드`, `대표번호`, `방문예약`을 노출하는 직진형 구조
- 사업위치, 규모, 입주예정처럼 고객이 먼저 확인하는 최소 정보만 상단에 요약하는 방식
- 이미지 슬라이더와 타입 탭을 이용해 사용자가 오래 고민하지 않게 만드는 간단한 탐색 구조
- 페이지 중후반을 길게 끌지 않고, `관심고객 등록 폼`으로 빠르게 전환시키는 설계
- 하단 고정 CTA와 반복 전화 유도 문구를 통한 즉시 문의 방식

### 참조 페이지에서 업그레이드할 요소

- 단순 나열형 구조를 `메디스파크 특화 가치` 중심의 설득형 구조로 강화
- 평면 이미지 위주의 정보 전달을 `입지-미래가치-실거주 장점` 순으로 재배치
- 대표번호를 `010-6689-2348`로 직접 노출해 모바일 즉시 전화 전환 강화
- 메디스파크 비전, 아주대병원, 교통망, 도보 생활권을 더 고급스럽게 스토리화
- 폼만 있는 구조에서 벗어나 `전화`, `문자`, `방문예약`, `상담신청`의 다중 CTA 구조로 확장

## 3. 반드시 반영할 유의사항

- PDF 하단 고지상 이 자료는 `분양직원 교육용 내부 자료` 성격이므로, 외부 랜딩페이지 반영 전 `문구/수치/개발 일정/착공 및 개원 예정 시점`은 반드시 최종 승인본으로 재검증해야 한다.
- 특히 `2026년 9월 개통 예정`, `2028년 착공`, `2030년 개원 예정`, `500병상`, `근무인원 1천명~1천5백명` 등은 홍보 심의 기준에 맞게 표현 수위를 조정해야 한다.
- 따라서 랜딩페이지 카피는 `예정`, `계획`, `비전`, `조성 추진`, `수혜 기대` 같은 표현 가이드를 함께 준비하는 것이 안전하다.

## 5. 터널 마케팅 전략

## 터널 마케팅의 핵심 해석

- 이 페이지의 목적은 많은 정보를 한 번에 보여주는 것이 아니라, 사용자가 빠져나가지 않고 `한 방향으로만` 전진하게 만드는 것이다.
- 따라서 상단부터 하단까지 아래 순서로 심리를 설계한다.

1. 멈추게 한다: 강한 첫 문장과 비주얼
2. 이유를 만든다: 왜 지금 봐야 하는지 입지와 개발 이슈 제시
3. 상상하게 한다: 실제 살기 좋은가를 단지와 평면으로 보여줌
4. 확신시킨다: 병원, 교육, 교통, 공원 등 객관 근거 제시
5. 행동하게 한다: 상담 신청 폼과 CTA 반복 노출

### 이 프로젝트에 맞는 터널 설계 포인트

- `브레인시티 안에서도 왜 메디스파크인가`를 가장 먼저 잡아야 한다.
- 단순 분양 광고가 아니라 `대형병원과 첨단 헬스케어 복합타운이 만드는 미래 생활권`이라는 큰 그림을 먼저 심는다.
- 이후 `교통`, `생활`, `단지`, `평면` 순으로 구체화하면서 실거주자와 투자 관심층을 동시에 설득한다.
- 페이지 중간중간 CTA를 끊어 넣어 정보 소비가 끝난 시점마다 바로 문의할 수 있게 한다.

## Section 2. 왜 지금 메디스파크인가

### 목적

- 사용자가 페이지를 계속 보게 만드는 가장 큰 이유를 준다.

### 구성

- 메디스파크 비전 이미지 활용: `hero_render/교육_11.jpg`
- 핵심 문장:
  - `브레인시티 안에서도 메디컬, 연구, 생활 인프라가 집중되는 핵심 축`
- 포인트 요약:
  - 아주대병원 인접 생활권
  - 메디스파크 및 의료 R&D 비전
  - 향후 유입 수요 기대
  - 의료, 연구, 직주근접 프리미엄 기대감

### 카피 방향

- 과장보다 `비전과 기대` 중심
- `의료 특화 복합타운이 만드는 생활권 프리미엄` 식의 표현 사용

### 참조 페이지 대비 업그레이드 포인트

- 참조 페이지는 단지 일반 홍보 중심이지만, 본 페이지는 `메디스파크`라는 이름 자체가 가진 차별성을 2번째 섹션에서 바로 고정시켜야 한다.
- 즉, 고객이 `브레인시티 단지 중 하나`가 아니라 `메디컬 비전 축의 핵심 단지`로 인식하도록 설계한다.

## Section 8. 상담 전환 구간

### 목적

- 페이지 하단에서 최종 전환을 유도한다.

### 구성

- 강한 문장:
  - `브레인시티 메디스파크 프리미엄, 분양 조건부터 확인해보세요`
- 혜택 문구 예시:
  - `관심고객 등록 시 잔여세대 및 분양 일정 우선 안내`
  - `방문예약 및 타입별 상담 가능`
- 대표번호 문구:
  - `빠른 문의 010-6689-2348`
- 입력 전 안내 문구:
  - `이름과 연락처만 남겨주시면 상담팀이 확인 후 빠르게 연락드립니다`
  - `분양가, 잔여세대, 방문예약 중 궁금한 내용만 편하게 문의하실 수 있습니다`
- 입력 항목:
  - 이름
  - 연락처
  - 관심타입
  - 상담유형
  - 개인정보 동의
- CTA:
  - `상담 신청하고 전화받기`
  - `상담 접수하기`
  - `문자상담 접수`

### 권장 상담 플로우

1. 사용자가 `상담하기` 버튼 클릭
2. 팝업 또는 하단 시트에서 `이름`, `연락처` 입력
3. 선택항목으로 `관심타입`, `문의내용`, `방문희망일` 입력 가능
4. `상담 신청하고 전화받기` 버튼 클릭
5. 프론트엔드에서 `Google Apps Script Web App URL`로 데이터 전송
6. Apps Script가 Google Sheets에 행 추가 저장
7. 접수 완료 후 문구 노출
   - `접수가 완료되었습니다. 담당자가 확인 후 순차적으로 연락드립니다.`
8. 필요 시 관리자 페이지 또는 CRM에서 즉시 콜백 진행

### 즉시 전화보다 접수형이 더 적합한 이유

- 사용자가 갑작스러운 통화보다 `먼저 남기고 기다리는 방식`에 심리적 부담이 덜하다.
- 상담팀은 이름과 연락처를 먼저 확보해 응대 품질을 높일 수 있다.
- 광고 유입 고객을 누락 없이 수집할 수 있어 후속 리마케팅에도 유리하다.

### DB 저장 항목 설계

- 필수 저장 항목:
  - `이름`
  - `연락처`
- 선택 저장 항목:
  - `관심타입`
  - `문의내용`
  - `방문희망일시`
- 운영 저장 항목:
  - `개발자정보`
  - `유입페이지명`
  - `프로젝트명`
  - `등록일시`
  - `상담상태`
  - `개인정보동의여부`

### 개발자 정보 저장 기준

- `개발자정보` 컬럼에는 고정값 `유정희`를 저장한다.
- 필요 시 `개발자명`, `버전`, `캠페인명`으로 세분화할 수 있으나 현재 기획 기준 기본값은 `유정희` 단일 저장으로 한다.

- 성공 응답 반환 후 프론트엔드에서 접수 완료 메시지 표시
